# 本地计费 / 支付联调（与 wework.syzs.top 对齐）

先在本机把计费页、微信/支付宝按钮、下单流程跑通，再打包上传 ECS。

## 地址对照

| 项 | 本地开发 | 生产 ECS |
|----|----------|----------|
| 前端 | `http://localhost:5173` | `https://wework.syzs.top` |
| 前端 API | 相对路径 `/api` → Vite 代理 **3000** | Nginx 反代 **3010**（`syqw-api`） |
| 后端进程 | `cd backend && npm run dev`（默认 **3000**） | PM2 `syqw-api`，`PORT=3010` |
| 静态资源 | Vite 内存 | `/var/www/wework/` |

**不要**在 `frontend/.env.production` 写 `localhost`；生产构建 `VITE_API_URL` 必须留空（同域 `/api`）。

## 1. 本机后端 `.env`（计费相关）

复制片段到 `backend/.env`（或对照 `backend/.env.billing.local.example`）：

```bash
PORT=3000
FRONTEND_ORIGIN=http://localhost:5173
FRONTEND_URL=http://localhost:5173
WEWORK_CALLBACK_URL=http://127.0.0.1:3000
BILLING_NOTIFY_BASE_URL=http://localhost:5173

# 本地显示「微信支付」按钮（模拟下单，无需商户号）
WECHAT_PAY_MOCK=1

# 支付宝：沙箱或 MOCK（二选一）
# ALIPAY_MOCK=1
# ALIPAY_DISABLED=0
# ALIPAY_APP_ID=...
# ALIPAY_PRIVATE_KEY_PATH=certs/alipay/app_private_key.pem
# ALIPAY_PUBLIC_KEY_PATH=certs/alipay/alipay_public_key.pem
```

验证：

```bash
cd backend
node --input-type=module -e "
import * as b from './src/services/billing.service.js';
console.log(JSON.stringify(b.getPaymentChannels(), null, 2));
"
```

期望：`wechat.enabled: true`（MOCK）、`alipay.enabled: true`（已配置或 MOCK）。

## 2. 本机数据库迁移（与线上一致）

```bash
cd backend
# 从 .env 读库账号
source <(grep -E '^DB_' .env | sed 's/^/export /')

mysql -h"$DB_HOST" -u"$DB_USER" -p"$DB_PASSWORD" "$DB_NAME" < ../database/095_payment_purchase_type.sql
mysql -h"$DB_HOST" -u"$DB_USER" -p"$DB_PASSWORD" "$DB_NAME" < ../deploy/ecs_fix_usage_addon_code.sql
```

若 `095` 文件不存在，执行仓库内 `deploy/ecs_fix_payment_metadata.sql` + `deploy/ecs_fix_usage_addon_code.sql`。

## 3. 启动顺序

```bash
# 终端 1
cd backend && npm run dev

# 终端 2
cd frontend && npm run dev
```

打开：`http://localhost:5173/app/billing`  
企业版应出现：**微信支付**、**支付宝**、线下转账、余额充值（与 channels API 一致）。

## 4. 生产为何没有微信支付？

生产 `getPaymentChannels()` 里 `wechat.enabled` 为 **false**（未配 `WECHAT_PAY_*` 且未开 MOCK）。  
要上真实微信：在 ECS `backend/.env` 配置商户号/证书，**不要**设 `WECHAT_PAY_MOCK=1`。

## 5. 上传生产 checklist

1. `npm run build`（`frontend/.env.production`：`VITE_API_URL=`、`VITE_ALIPAY_ENABLED=1`）
2. `rsync` 到 `/var/www/wework/`
3. 同步 `backend/` 到 `/var/www/wework-saas/backend/`
4. ECS 执行：`deploy/ecs_fix_payment_metadata.sql`、`deploy/ecs_fix_usage_addon_code.sql`
5. `pm2 restart syqw-api --update-env`
6. 确认 Nginx `proxy_pass` → **3010**，`grep proxy_pass /etc/nginx/conf.d/wework-https.conf`

## 6. 常见错误

| 报错 | 处理 |
|------|------|
| `Data truncated for column 'purchase_type'` | 跑 `deploy/ecs_fix_payment_metadata.sql` 修正 ENUM |
| `Unknown column 'metadata'` | 同上（095 迁移） |
| `Unknown column 'code'`（addons） | 跑 `deploy/ecs_fix_usage_addon_code.sql` |
| 线上 CORS 连 localhost | 重建前端，勿在 production env 写 localhost |
| 支付宝 DECODER | ECS 内联 PEM 或上传新版 `env.js` + `ALIPAY_*_PATH` |
