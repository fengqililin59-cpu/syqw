# Post-release 30-minute Observation Checklist

> 用途：发布后 30 分钟内的稳定性观察，尽早发现回归。  
> 建议发布负责人每 5 分钟检查一次并记录。

## 0-5 分钟（基础可用性）

- [ ] `GET /health` 正常
- [ ] `GET /health?deep=1` 正常（`database=true`）
- [ ] 登录页、注册页可打开
- [ ] 后台首页可加载

## 5-10 分钟（核心业务）

- [ ] 登录成功率正常（人工或监控）
- [ ] 注册流程可完成
- [ ] `渠道分析` 页面可打开并查询
- [ ] `广告ROI` 页面可打开并查询

## 10-20 分钟（接口与依赖）

- [ ] `/api/v1/track/report` 成功返回
- [ ] `/api/v1/ads/roi` 成功返回
- [ ] `/api/v1/ads/roi/trend` 成功返回
- [ ] `/api/v1/ads/roi/details` 成功返回
- [ ] AI 文案/海报接口响应正常（若本次涉及）

## 20-30 分钟（质量与性能）

- [ ] 无持续 5xx 峰值
- [ ] 核心接口 P95 无明显恶化
- [ ] PM2 进程无异常重启飙升
- [ ] Nginx error log 无持续新增严重错误
