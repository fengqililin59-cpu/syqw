# 让消费者在 AI 大模型里搜到你的品牌、产品与公司

> **说明**：ChatGPT、豆包、通义、文心、Kimi 等不会单独给你「付费上榜」开关。用户问「XX 是什么 / 哪家好」时，模型主要依据**公开网页、百科、新闻、社区讨论、结构化数据**做归纳。下面分 **ZhiFlow 产品内可做** 与 **站外必做** 两部分。

## ZhiFlow 里与「可被找到」相关的功能

| 能力 | 菜单位置 | 作用 |
|------|----------|------|
| 营销落地页 | 部署后的 `/landing.html`（根路径静态页） | 品牌介绍、价值主张；需被搜索引擎与爬虫可访问 |
| H5 留资页 | `/lead-form.html?tenant=…` | 投流落地、带 UTM；经 **获客向导** 监测链可带 `ad_hit` |
| 获客向导（监测 URL） | **获客向导** `/app/acquisition-wizard` | 各广告平台监测链、留资链、转化说明 |
| 广告 ROI / 渠道 | **广告 ROI** `/app/ads-roi`、**渠道分析** `/app/channel-report` | 验证投流线索与转化，与「被 AI 提到」无直接关系但保证投放闭环 |
| AI 知识库（对内 RAG） | **AI 知识库** `/app/knowledge-base` | 供收件箱/AI 员工回复客户；**默认需登录**，不直接等于公网 SEO |
| 知识库管理 | **知识库管理** `/app/kb-articles` | 维护文章、发布状态、slug；发布后可在公开帮助中心展示 |
| 公开帮助中心 | `/help-center.html?tenant=租户ID` | 无需登录，展示已发布分类与文章；挂到官网导航便于爬虫与 AI 收录 |
| 租户品牌页（JSON-LD） | `/brand.html?tenant=租户ID` | `Organization` + `SoftwareApplication` 结构化数据 |
| 使用帮助 | **使用帮助** `/app/help` | 租户内上手文档 |
| AI 员工 playbook | **AI 员工** 相关向导 `/app/ai-employee-playbook` | 配置知识库 + 自动跟进闭环 |

**公开帮助中心（已上线）**

| 类型 | URL / 接口 |
|------|------------|
| 帮助中心页面 | `https://你的前端域名/help-center.html?tenant={租户ID}` |
| 单篇文章 | `…/help-center.html?tenant={租户ID}&article={slug}` |
| 品牌 JSON-LD 页 | `https://你的前端域名/brand.html?tenant={租户ID}` |
| 公开 API | `GET /api/v1/kb/public/categories?tenant=`、`GET /api/v1/kb/public/articles?tenant=`、`GET /api/v1/kb/public/articles/{slug}?tenant=` |
| 租户品牌信息 | `GET /api/v1/public/tenants/{租户ID}/branding`（仅返回名称与公开链接，不含密钥） |

在 **知识库管理** 发布文章并填写 `slug` 后，将帮助中心链接放到官网、落地页页脚或百科引用处。分类需在知识库中勾选「已发布」。GEO 仍建议配合 **自有官网 + 站外百科**。

## 实操清单（GEO / AEO：生成式引擎优化）

按优先级执行，约 5–10 步即可起步：

1. **独立官网或子域**  
   使用清晰品牌名、产品名、公司全称各出现多次（标题、H1、首段、页脚）。部署 `landing.html` 或自建站，保证 **HTTPS、可被爬取**（勿整站登录墙）。

2. **撰写「可被引用」的说明页**  
   在 ZhiFlow **知识库管理** 写：公司是谁、解决什么问题、适用行业、价格区间（如可公开）、与竞品差异。语气客观、带小标题，便于模型摘录。发布后同步到官网「关于我们 / 产品」栏目。

3. **结构化数据（Schema.org）**  
   在官网加入 `Organization`、`Product` 或 `SoftwareApplication` 的 JSON-LD（名称、url、logo、description）。帮助搜索引擎与部分 AI 索引理解实体。

4. **百度百科 / 企查查 / 天眼查**  
   企业主体、品牌名、产品名保持一致；百科词条与官网描述互相引用（注意百科中立语气与来源要求）。

5. **知乎、公众号、行业媒体露出**  
   用真实案例回答「XX 行业 CRM / 私域工具怎么选」类问题，文中自然出现品牌与产品名（非堆砌关键词）。与 **知乎投流** 监测可并用：获客向导 → 知乎监测 URL。

6. **产品文档与 FAQ 可爬取**  
   将帮助文档放在 **无需登录** 的 `/help` 或博客子路径；`robots.txt` 勿误拦；提供 `sitemap.xml`。

7. **一致的品牌实体**  
   各平台账号名、官网 title、知识库标题、工商注册名尽量统一，减少模型把你们拆成多个实体。

8. **允许 AI 爬虫（按需）**  
   在 `robots.txt` 中了解 GPTBot、Bytespider 等；若希望被收录可明确 `Allow`（同时评估内容敏感性）。

9. **投流 + 私域闭环（ZhiFlow）**  
   **获客向导** 配置监测链 → 留资/加企微 → **广告 ROI** 看回传。对外曝光与广告数据分开，但优质落地页内容会提高转化与品牌搜索量。

10. **监测品牌词**  
    定期在豆包、通义、文心、ChatGPT 中问：「[你的品牌] 是什么」「[品类] 有哪些工具」，记录是否提及、描述是否准确；不准确则补充官网与百科并增加第三方可信来源。

## 与「广告对接」的关系

- **广告平台**（抖音、腾讯、百度、快手、小红书、**知乎** 等）：在 **获客向导** 复制监测 URL，在各自广告后台投放；转化在 **广告 ROI** 查看。  
- **AI 大模型收录**：主要靠 **公网内容与实体一致性**，不是广告后台一项设置。
