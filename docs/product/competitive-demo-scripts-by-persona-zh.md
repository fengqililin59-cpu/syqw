# ZhiFlow 分角色演示脚本（5 分钟版）

> 与 [飞书话术表](./competitive-sales-feishu-table-zh.md) 六类客户一一对应；售前按「客户类型」选章，勿混讲模块顺序。  
> 通用环境准备见 [demo-script.md](./demo-script.md)；本节仅列各角色**必查项**与**差异化话术**。

---

## 1. 投流型中小企业

### 演示前必查

- [ ] 广告 ROI / 监测链有至少 2 条渠道样例数据  
- [ ] 客户列表含高意向、待跟进各 ≥3 条  
- [ ] 「本周战果」看板有本周跟进/转化数字  
- [ ] `/health?deep=1` 返回 200  

### 30 秒开场

> 「您现在最头疼的是投流钱花在哪、销售跟没跟高意向吧？ZhiFlow 在企微里把留资、加微和跟进串起来，意向分告诉您先跟谁，AI 只出草稿、您点一下才发。我们用 14 天看广告 ROI 和本周战果，不看功能表谁最长。」

### 5 分钟演示（点哪里 · 说什么）

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | 打开 **广告 ROI / 获客监测链** | 「这条渠道留了多少、加微多少、跟到哪一步，一条链看清，不用导三张表。」 |
| 1:30 | **客户列表** → 筛「高意向 / 预警」 | 「系统按行为打分，销售不用翻几百个好友，先跟这几个人。」 |
| 2:30 | 点一条客户 → **待跟进** 或跟进任务 | 「漏跟会标出来，老板在手机也能看见谁该动。」 |
| 3:30 | **收件箱** → 选会话 → 展示 AI 草稿 → **审核发出** | 「话术 AI 写好，对外必须您点发送，不会自动乱回客户。」 |
| 4:30 | **本周战果** 看板 | 「老板看这一周跟了多少、转化多少，和投流账对得上。」 |

### 收尾一问

> 「方便先开 14 天专业版试用吗？我们帮您把本周一条主渠道 ROI 和战果跑通，下周一起对数。」

## 2. 已有尘峰 / 微伴

### 演示前必查

- [ ] 话术避免「替换尘峰/微伴」表述，强调「补充」  
- [ ] 高意向预警、收件箱草稿流程顺畅  
- [ ] 广告 ROI 有数据（作第四幕「增量价值」）  

### 30 秒开场

> 「尘峰/微伴在存档、营销工具上很全；很多 10–30 人团队仍漏跟、AI 不敢开。ZhiFlow 可以只补意向预警、收件箱 AI 草稿人发和投流算账，不必推倒现有系统。」

### 5 分钟演示

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | **仪表盘 / 高意向预警** | 「在现有流程上加一层：今天谁先跟，系统推给您。」 |
| 1:30 | 点预警客户 → **AI 跟进话术**（复制/人发） | 「省写话术时间，发送权仍在销售手里。」 |
| 2:30 | **收件箱** → 草稿 → **审核发出** | 「和贵司现有群发不冲突，这是 1v1 会话里的草稿+人审。」 |
| 3:30 | 同客户 **时间线 / 意向分变化** | 「跟进记录集中看，交接不用翻多个模块。」 |
| 4:30 | **广告 ROI**（简要） | 「若贵司也在投流，这块是尘峰/微伴里常要单独补的算账视角。」 |

### 收尾一问

> 「能否先按补充场景开 14 天试用，只验证意向+收件箱+ROI 三块增量？满意再谈席位，不动主系统合同。」

## 3. 有赞零售转型（多渠道 B2B 线索）

### 演示前必查

- [ ] 演示数据偏 **B2B 线索 / 阶段**，勿主打门店订单、分佣  
- [ ] 线索进库、阶段 pipeline 已配置  
- [ ] 广告 ROI 展示「获客」非「门店报表」  

### 30 秒开场

> 「有赞在卖货和会员上很强；若您多渠道线索进企微、要顾问式跟单、还要算清获客 ROI，ZhiFlow 贴这一块。零售履约仍建议有赞主战场，我们补投流+销售转化。」

### 5 分钟演示

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | **线索进库** → 看来源与阶段 | 「抖音/表单/活码进来的线索，统一进库，不靠导购 SOP。」 |
| 1:30 | **意向分 / 待跟进** 列表 | 「顾问式销售看分数和待办，不是门店客流那一套。」 |
| 2:30 | 客户详情 → **跟进时间线** | 「电话、企微、备注一条线，换人跟单不断档。」 |
| 3:30 | **收件箱** → AI 草稿 → 人审发出 | 「复杂产品介绍，AI 起草，顾问改一句再发。」 |
| 4:30 | **广告 ROI** | 「算的是获客与跟进转化，不是店铺 GMV 报表。」 |

### 收尾一问

> 「能否用 14 天试一条「投流线索 → 企微跟进 → ROI」闭环？和有赞并行，不替换店铺系统。」

## 4. 老板顾虑 AI

### 演示前必查

- [ ] 租户/平台 **AI 开关** 位置确认，可演示「关」  
- [ ] 收件箱仅展示 **草稿 → 审核发出**，无自动对外发送  
- [ ] 话术材料备「阶段 B 半自动需单独协议」一句（若被问）  

### 30 秒开场

> 「您担心 AI 乱答、合规—we 默认草稿+人审，公域只出草稿；半自动仅白名单试点，平台可一键关停。定位是销售副驾驶：省写、少漏跟，不是替代客服部。」

### 5 分钟演示

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | **设置 / AI 开关**（租户或平台说明） | 「老板您能看见开关；关了就只人工，没有后台偷偷发。」 |
| 1:00 | **收件箱** → 新消息 → 展示 **仅草稿** | 「客户消息进来，AI 只填草稿框，不直接进客户微信。」 |
| 2:00 | 编辑草稿 → 点击 **审核/发送** | 「每一句对外都是您或销售点过才出去。」 |
| 3:00 | **客户详情** → AI 话术建议 → **复制后人发** | 「1v1 场景同样：建议可采纳，发送权在人。」 |
| 4:00 | 简要 **意向/待跟进**（30 秒） | 「AI 还帮您排优先级，但不替您开口对客户承诺。」 |
| 4:30 | 重申：无「全自动客服」承诺 | 「我们不做替代 80% 客服；要半自动须单独试点协议。」 |

### 收尾一问

> 「建议老板本人或合规负责人开 14 天试用，只走「草稿→您点发送」一条路径；满意再开席。」

## 5. 和扣子组合

### 演示前必查

- [ ] 不贬低扣子；集成边界售前已口头对齐（API 范围）  
- [ ] 客户列表、时间线有完整样例客户  
- [ ] 收件箱审核发信流程可用  

### 30 秒开场

> 「扣子擅长造 Agent 和复杂对话；ZhiFlow 管客户资产、跟进度、审核发出和 ROI。复杂问答走扣子，客户在哪、跟到哪、谁在跟，在 ZhiFlow 闭环。」

### 5 分钟演示

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | **客户列表** → 点一名客户 | 「扣子对话里的意向，落到可跟进的客户档案。」 |
| 1:30 | **时间线**（跟进、阶段变更） | 「销售每天打开的是进度，不是散落的 Bot 记录。」 |
| 2:30 | **收件箱** → 草稿 → **审核发出** | 「触达走企微，必须人审；和扣子 API 对接即可。」 |
| 3:30 | **意向分 / 待跟进** | 「Bot 聊得再热，这里告诉您要不要销售介入。」 |
| 4:30 | **广告 ROI / 本周战果** | 「从投放到成交，数据在 CRM 侧汇总，不只在对话里。」 |

### 收尾一问

> 「能否 14 天试用 ZhiFlow 专业版，同时我们售前一起确认和扣子 API 的对接清单？席位按组合方案评估。」

## 6. 平台 / 售前演示前（对内）

### 演示前必查

- [ ] `/health?deep=1` = **200**  
- [ ] 演示账号为 **销售角色**，菜单无平台/他租户泄漏  
- [ ] **租户迁移**已在该演示租户执行  
- [ ] 模块顺序背熟：高意向 → AI 话术 → 收件箱草稿 → 广告 ROI → 本周战果  
- [ ] 对外口径：阶段 A、单租户试点；不提未验收环境  

### 30 秒开场（对客户）

> 「今天给您看销售在企微里的闭环：先跟谁、怎么写、您来发、投流怎么算账。我们按试点单租户推进，功能以您环境验收为准。」

### 5 分钟演示（严格顺序）

| 分钟 | 操作 | 话术要点 |
|------|------|----------|
| 0:30 | **高意向 / 预警** | 「第一屏：今天优先跟谁。」 |
| 1:30 | 点客户 → **AI 话术**（人发/复制） | 「第二屏：怎么开口，AI 建议、人决定。」 |
| 2:30 | **收件箱** → 草稿 → **审核发出** | 「第三屏：对外发送必须审核。」 |
| 3:30 | **广告 ROI** | 「第四屏：投流与跟进是否对上账。」 |
| 4:30 | **本周战果** | 「第五屏：老板一眼看本周产出。」 |

### 收尾一问

> 「报价锚定专业版 ¥299/月起 + 14 天试用；下一步安排迁移与 deep health 验收后再扩席，您看是否先定试点租户？」
