# ZhiFlow 竞争优势提升路线图（30 / 60 / 90 天）

> 目标：把「功能多但 production 脆、菜单宽但 polish 不足」收敛成 **可卖、可演示、可续费** 的差异化。  
> 基线文档：[competitive-positioning-vs-scrm-zh.md](./competitive-positioning-vs-scrm-zh.md) · [competitive-moat.md](./competitive-moat.md)

## 当前真实短板（对话与运维共识）

| 短板 | 对销售/续费的影响 | 优先级 |
|------|-------------------|--------|
| 生产库 **迁移未齐**（072–076+ 业务表） | 新功能 500、权限缺列 | **P0** |
| 公网 **502** / PM2 端口 / Nginx 上游不一致 | 无法演示、信任崩塌 | **P0** |
| **RBAC 与菜单** 不一致（销售见平台菜单或缺收件箱） | 售前演示翻车 | **P0** |
| **菜单面宽、核心路径 polish 不足** | 竞品对比显得「半成品」 | **P1** |
| 品牌 **GEO/声量** 弱于尘峰/微伴/有赞 | 线索靠人肉介绍 | **P1** |
| 会话存档/飞单/AI 全自动客服 | 大单招标吃亏 | **P2**（明确不主打） |

## 30 天（本月必须）：「能稳定卖、能稳定演示」

### 1. 稳定性（生产）

| # | 动作 | 完成标准 | 负责人建议 |
|---|------|----------|------------|
| 1.1 | 生产执行 **迁移清单**至最新（至少 072–076；计费/支付按需 043/062/064/065） | `schema_migrations` 无缺口；`SHOW TABLES` 关键表存在 | 运维 + 后端 |
| 1.2 | 固化 **发布 SOP**：`./scripts/deploy-check.sh` → PM2 reload → `curl /health?deep=1` | 每次发布有记录；公网 deep health **200** | 运维 |
| 1.3 | 消灭 **502**：ECS 上 `zhiflow-api` online，Nginx `proxy_pass` **127.0.0.1:3002** | 连续 7 天无 upstream 502 | 运维 |
| 1.4 | 阶段 A 环境：`INBOX_AI_AUTO_SEND=0`，`DEEPSEEK_API_KEY` 有效，关闭支付 Mock | [go-live-ai-inbox](../deploy/go-live-ai-inbox.md) 30 分钟验收通过 | 产品 + 运维 |
| 1.5 | 开启 **cron**：高意向预警、SLA、本周战报（按 env 文档） | 试点租户能收到企微提醒 | 运维 |

### 2. 核心闭环（投流归因 + AI 收件箱）

| # | 动作 | 完成标准 |
|---|------|----------|
| 2.1 | 选 **1 个行业、1 条监测链** 跑通：获客向导 → 留资/加微 → 广告 ROI 有数 | 销售能讲清「花了多少钱、来了多少线索」 |
| 2.2 | 收件箱 **黄金路径** 打磨：同步 → 草稿 → 审核 → 发出 → 线程写回客户时间轴 | 演示 5 分钟无卡顿、无 403 |
| 2.3 | 仪表盘固定 **「每日 3 步」** 入口（待跟进 / 收件箱 / AI 话术）置顶 | 新销售 3 天内会用（见 [quickstart-3min.md](./quickstart-3min.md)） |

### 3. 权限与菜单（修「弱」项）

| # | 动作 | 完成标准 |
|---|------|----------|
| 3.1 | 跑通 [permission-rbac-audit-checklist.md](../permission-rbac-audit-checklist.md) + API 脚本 | 销售/管理员/平台角色 **零交叉泄漏** |
| 3.2 | `DashboardLayout`：**默认角色菜单** 与 `requirePerm` 对齐 | 销售只见 客户/收件箱/仪表盘/帮助；不见平台 |
| 3.3 | 新租户 onboarding：勾选收件箱+服务台权限模板 | [onboarding.md](./onboarding.md) 一次配置成功 |

### 4. GTM / 定价（轻量）

| # | 动作 | 完成标准 |
|---|------|----------|
| 4.1 | 销售包：**对比表 + 话术** 进 CRM/飞书（本文档 § 销售话术） | 全员同一口径 |
| 4.2 | 落地页 `/landing.html` 增加 **vs SCRM** 锚点（3 行：意向/人审/ROI） | 与 competitive-positioning 一致 |
| 4.3 | 试点 **3–5 家** 阶段 A 客户，每周复盘：登录天数、AI 草稿数、跟进数 | 有 1 个可复述案例 |

## 60 天：「差异化可感知、竞品难一句带过」

### 差异化深化

| # | 动作 | 完成标准 |
|---|------|----------|
| 5.1 | **行业 playbook** 在 3 个垂直各落 1 个完整案例（教培/美业/B2B） | 含：话术包 + 预警 + 战报截图 |
| 5.2 | **收件箱 ↔ CRM 阶段** 联动：演示「改阶段即出现在待跟进」 | 销售感到「一个系统」 |
| 5.3 | **本周战果** 企微推送 + 管理员口述「本月省 X 小时」模板 | 续费谈话有数据 |
| 5.4 | 广告 ROI：**知乎/小红书/抖音** 至少 2 个平台回传在试点租户验证 | 对标尘峰「能回传」叙事 |
| 5.5 | 菜单 **收敛**：非核心项收进「更多」或管理员开关 | 新用户菜单项 ≤ 竞品一线销售常用集 |

### 稳定性与质量

| # | 动作 | 完成标准 |
|---|------|----------|
| 6.1 | GitHub **CI 绿** + 发布前 `backend npm test` 门禁 | main 分支可发布 |
| 6.2 | **发布后 30 分钟** checklist 制度化（[post-release-30min](../ops/checklists/post-release-30min.md)） | 两次发布零回滚 |
| 6.3 | 平台 **MRR / churn 预警** 周会看板 | 体验到期前 7 天干预 |

### GTM / GEO

| # | 动作 | 完成标准 |
|---|------|----------|
| 7.1 | 执行 [ai-brand-discoverability-zh.md](./ai-brand-discoverability-zh.md) 前 5 步 | 帮助中心 + brand.json 可爬；官网一致实体名 |
| 7.2 | 知乎/公众号 **1 篇**「企微 SCRM 怎么选」客观文，带 ZhiFlow 场景 | 品牌词在 2 个大模型里可被搜到（记录截图） |
| 7.3 | **演示环境** 独立子域 + 只读样板数据 | 不再用生产租户演示 |

## 90 天：「可规模化获客、可选阶段 B」

### 核心闭环进阶

| # | 动作 | 完成标准 |
|---|------|----------|
| 8.1 | **阶段 B**：3–5 家白名单 `INBOX_AI_AUTO_SEND` FAQ/询价半自动 | 平台 AI 异常周回顾；零重大投诉 |
| 8.2 | **订单/回款** 与广告 ROI 打通（至少手工订单关联渠道） | 管理者看「线索→成交」漏斗 |
| 8.3 | 员工活动 **轻量看板**（跟进数/响应时长排行） | 对标「老板知团队在干嘛」弱点（见 [ai_feature_gap_analysis.md](../ai_feature_gap_analysis.md)） |

### 定价与包装

| # | 动作 | 完成标准 |
|---|------|----------|
| 9.1 | 对外 **三档故事**：体验版 / 专业版（销售闭环）/ AI 加购 | 价目表与 [pricing.md](./pricing.md) 一致 |
| 9.2 | **年框/合同开单** 模板在平台跑通 2 单 | 中大客户不必只靠月付 |
| 9.3 | 兑换码 + 渠道备注追踪 | 投放可算 CAC |

### 明确不做或只做调研（避免 scope 爆炸）

- 全套 **会话存档取证**（对标尘峰）— 仅调研集成，不承诺 90 天内自研  
- **AI 全自动替代 80% 客服** — 仅白名单；对外仍主打人审  
- **AI 投流自动出价** — P2 以后  
- **招聘/AI 财务** — 不做

## 按类别总览

| 类别 | 30 天 | 60 天 | 90 天 |
|------|-------|-------|-------|
| **稳定性（生产）** | 迁移齐、502 零、health、cron | CI 门禁、发布 SOP | 演示环境独立 |
| **核心闭环** | 一条监测链 ROI + 收件箱黄金路径 | playbook 案例、阶段联动 | 阶段 B 白名单、线索→成交 |
| **差异化深化** | 每日 3 步、战报可见 | 行业案例、菜单收敛 | 员工活动看板 |
| **GTM / GEO** | 统一话术、3–5 试点 | 知乎文+帮助中心 | 品牌词监测常态化 |
| **定价包装** | 试点续费谈话模板 | 落地页对比锚点 | 年框+兑换码追踪 |

---

## 成功指标（建议周看）

| 指标 | 30 天目标 | 90 天目标 |
|------|-----------|-----------|
| 生产 `deep health` 可用率 | ≥ 99.5% | ≥ 99.9% |
| 试点租户周活跃（≥3 天登录） | ≥ 60% | ≥ 70% |
| 试用租户「AI 草稿 + 跟进」双激活 | ≥ 50% | ≥ 65% |
| 可对外案例（有 ROI/战报数字） | 1 个 | 3 个 |
| 权限相关支持工单 | 环比下降 80% | 趋近 0 |
