# ZhiFlow 竞争壁垒与「离不开」策略

## 一、护城河从哪来（相对微伴 / 扣子 / 表格）

| 壁垒类型 | ZhiFlow 做法 | 用户感知 |
|----------|--------------|----------|
| **工作流嵌入** | 待跟进 → AI 写话术 → 看板更新 → 企微人发 | 每天打开的理由 |
| **数据资产** | 客户、跟进、意向分、会话上下文沉淀在租户库 | 迁移成本高 |
| **安全合规** | 默认 AI 草稿 + 人审；可选企微 FAQ/询价半自动（护栏 + 抽检 + 平台关停） | 老板敢开、出问题可关 |
| **可量化价值** | 仪表盘效率概览 + 每周战报 | 续费有理由 |
| **企微原生** | 活码、回调入库、应用消息提醒 | 不用切 5 个工具 |

扣子强在「造 Agent」，微伴强在「功能 breadth」——ZhiFlow 应打 **「销售每天少写 30 分钟话术、少漏跟 10 个高意向」**，而不是功能表更长。

## 二、提高单用户价值（AARRR 中的 Retention / Revenue）

1. **激活（首日）**  
   注册 → AI 首问弹窗 → 客户详情 AI 话术（已实现）  
   目标：7 日内 `ai_calls_used > 0` 且 `follow_ups` 有记录  

2. **习惯（首周）**  
   销售「每日 3 步」+ 待跟进企微提醒  
   目标：每周至少 3 天登录并完成 1 次跟进登记  

3. **证明（首月）**  
   仪表盘 ROI + **每周价值战报**（企微推送给管理员）  
   目标：管理员能说出「本月 AI 省了 X 小时」  

4. **扩展（付费）**  
   试用将到期提醒 → 微信支付 / 兑换码  
   AI 配额触顶 → AI 专用套餐加购  

5. **锁定（团队）**  
   角色权限、审计日志、客户归属、流程编排  
   目标：换工具 = 重配流程 + 丢历史上下文
