# ZhiFlow vs 主流企微 SCRM 竞品定位（销售用）

> **诚实原则**：本表基于公开资料、常见投标应答与 ZhiFlow 当前代码/文档能力（2026-06）。「赢/平/弱」指**可对外承诺的成熟度**，不是愿景。  
> 相关：[竞争壁垒](./competitive-moat.md) · [功能清单](./features.md) · [定价](./pricing.md) · [扣子对比](./coze-comparison.md)

## 竞品范围

| 简称 | 代表产品 | 典型买家 |
|------|----------|----------|
| **尘峰** | 尘锋 SCRM / 营销云 | 20+ 人销售团队、重营销自动化与合规存档 |
| **微伴** | 微伴助手 / 微盛·企微管家 | 10–50 人团队、要「功能全、上手快」 |
| **有赞/微盟** | 有赞企微助手、微盟智慧导购等 | 零售/本地生活、强交易与门店 |

ZhiFlow 定位：**企微私域里的「销售副驾驶」**——意向谁该跟、AI 怎么说、**人来发**；投流可归因、价值可量化。不打「功能表最长」。

## 功能维度对比（赢 / 平 / 弱）

图例：**赢** = 可主动打差异化且交付稳定 · **平** = 均有，差异在深度/价格/实施 · **弱** = 对方明显更强或我方未达可售成熟度

| 维度 | ZhiFlow | 尘峰 | 微伴 | 有赞/微盟 | 判定 |
|------|---------|------|------|-----------|------|
| **企微客户同步 / 活码 / 继承** | 活码、回调、继承 | 成熟 | 成熟 | 成熟（偏门店） | **平** |
| **客户画像 / 阶段 / 标签 / 跟进** | CRM 核心闭环 | 深 | 深 | 中（偏会员） | **平**（深字段/报表对方更全） |
| **群发 / SOP / 自动化流程** | 有 React Flow 编排 | 强 | 强 | 强（营销插件多） | **平**（模板生态对方更厚） |
| **会话存档 / 合规质检** | 消息入库、AI 审核台 | **行业标杆级** | 强 | 中 | **弱** |
| **侧边栏 / 一线销售体验** | 有，菜单多、打磨中 | 成熟 | 成熟 | 导购场景深 | **弱**（广而欠 polish） |
| **AI 意向分 + 高意向预警** | 规则+AI 混合、企微提醒 | 有类似 | 有类似 | 弱 | **赢**（与「每日 3 步」绑定紧） |
| **统一收件箱 + AI 草稿 + 人审** | 默认草稿；半自动 FAQ 试点 | 客服模块强 | 客服/机器人强 | 偏店铺客服 | **赢**（合规叙事：敢开、可关） |
| **知识库 RAG（对内）** | 向量+关键词、收件箱用 | 有 | 有 | 商品库更强 | **平** |
| **投流监测 / 多平台转化回传** | 获客向导 + 广告 ROI | 有 | 有 | 有（偏电商投放） | **平**（我方闭环完整但**生产稳定性**须先过关） |
| **订单 / 工单 / 售后** | 服务台 + 收件箱转工单 | 有 | 有 | **交易闭环最强** | **弱**（零售/O2O 选有赞） |
| **电话外呼 / 短信** | TCCC + 阿里云 SMS | 常打包 | 常打包 | 短信/导购触达 | **平**（需客户另开第三方） |
| **数据看板 / 管理者 ROI** | 效率概览、本周战果、战报 | 报表多 | 报表多 | 经营报表强 | **平**（我方强调「AI 省多少时间」故事） |
| **行业话术包 / Playbook** | 教培/美业/B2B 已上线 | 模板多 | 模板多 | 零售模板多 | **平**（我方更贴 AI+跟进闭环） |
| **RBAC / 审计** | 细粒度权限（**偶有菜单/权限不一致**） | 成熟 | 成熟 | 角色简单 | **弱**（须修到可演示零尴尬） |
| **定价透明度 / 中小团队门槛** | ¥299/月起、14 天试用 | 通常更高、商务议价 | 席位+模块 | 绑店铺套餐 | **赢**（10 人以内销售团队） |
| **私有化 / 数据出域** | 企业版支持 | 支持 | 部分支持 | 云为主 | **平** |
| **生产就绪 / 实施风险** | **迁移、502、权限需收敛** | 成熟交付体系 | 成熟 | 成熟 | **弱**（对内必说，对外用阶段 A 口径） |
| **品牌/GEO（AI 搜到）** | 帮助中心、brand JSON-LD 已做 | 声量大 | 声量大 | 声量大 | **弱** |

### 汇总：我们适合主动打的仗

1. **「AI 写草稿，销售点一下才发」** — 老板敢开、出问题平台可关停（对比竞品「全自动客服」承诺过度）。  
2. **「谁该跟」** — 意向分 + 预警 + 待跟进，不是又一个群发工具。  
3. **「投流不是黑洞」** — 监测链 → 留资/加微 → ROI 看板（交付前强调**环境已跑通迁移与健康检查**）。  
4. **「中小团队买得起的专业版」** — 对比尘峰/微伴商务盘。  
5. **「和扣子组合」** — 复杂 Bot 用扣子，客户资产与审核发信在 ZhiFlow（见 [coze-comparison.md](./coze-comparison.md)）。

### 汇总：应回避或诚实说明的仗

- 大型集团 **会话存档、合规取证、飞单防控** 招标 → 尘峰/微伴更合适，或 ZhiFlow 作补充。  
- **连锁零售、订单履约、导购分佣** → 有赞/微盟。  
- **「替代 80% 客服人力、全自动」** → 我方仅阶段 B 白名单试点，见 [go-live-ai-inbox.md](../deploy/go-live-ai-inbox.md)。

## 销售话术 Bullets（按场景）

### 电梯稿（30 秒）

「ZhiFlow 是给企微销售团队用的私域增长系统：客户自动进库、AI 算意向告诉你先跟谁，收件箱里 AI 写好草稿、你审核后才发给客户。投流线索能归因到 ROI，管理员每周能看到 AI 帮团队省了多少时间。我们不是功能最多的 SCRM，而是**让销售每天少写半小时、少漏跟十个高意向**。」

### vs 尘峰

- 「尘峰适合已经很大、很看重存档合规和营销全家桶的团队；ZhiFlow 更适合 **10–30 人销售团队**，要 **AI 跟进 + 人审发信 + 意向预警** 快速见效，**月费透明、两周试用**。」  
- 「若您已有尘峰，我们可谈 **收件箱 AI 草稿、意向分、广告 ROI** 作补充，而不是推倒重来。」

### vs 微伴 / 微盛

- 「微伴菜单多、功能全；我们刻意做 **销售每日闭环**：待跟进 → AI 话术 → 人发 → 看板更新，减少销售在系统里迷路。」  
- 「AI 默认 **草稿+人审**，公域渠道只草稿，降低乱答风险；适合老板对 AI 客服有顾虑的团队。」  
- 「诚实说：群发模板库、会话质检深度我们还在追；若您最看重 **存档取证**，请以尘峰/微伴为主评。」

### vs 有赞企微 / 微盟

- 「有赞强在 **卖货和会员运营**；ZhiFlow 强在 **B2B/顾问式销售、多渠道线索跟进、AI 销售辅助**。零售主战场建议有赞；若您是 **投流获客 + 企微销售转化**，我们更合适。」

### 客户问「你们和 XX 比功能少吗？」

「对，我们不做最长功能表。我们赌的是三件事：**高意向不漏跟、AI 帮你写但你来发、投流能算清账**。您可以 14 天试用，用 **本周战果** 和 **广告 ROI** 看值不值。」

### 客户问「生产稳不稳？」（对内必准备）

「我们处于 **阶段 A 推广**：核心 API 与 AI 草稿已上线，上线前会与您一起完成 **数据库迁移、健康检查与权限验收**（见 [production-launch-runbook-zh.md](../deploy/production-launch-runbook-zh.md)）。试点客户优先 **单租户白名单**，稳定后再扩席位。」

### 与扣子组合（见 coze-comparison）

「扣子 **造 Agent**；ZhiFlow **管客户、跟进度、审核发出、算 ROI**。两个 API 对接即可。」

## 演示建议（销售/售前）

| 顺序 | 演示点 | 话术锚点 |
|------|--------|----------|
| 1 | 客户列表 → 高意向 / 预警 | 「系统告诉你先跟谁」 |
| 2 | 客户详情 → AI 跟进话术 | 「30 秒写好，你来发」 |
| 3 | 统一收件箱 → 草稿 → 审核通过 | 「老板敢开的 AI」 |
| 4 | 广告 ROI / 获客向导监测链 | 「投流不是黑洞」 |
| 5 | 仪表盘「本周战果」 | 「续费理由」 |

**演示前检查**：`/health?deep=1` 200、销售角色权限无多余/缺失菜单、试用租户已跑迁移（避免现场 502/空白页）。
