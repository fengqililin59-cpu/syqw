# ZhiFlow vs 扣子（Coze）— 对外材料

> 内部用：销售、官网、投标可各取一节。最后更新：2026-05。  
> 官网一屏对比：`/landing.html#compare`

## 一句话

**扣子是「造 AI」；ZhiFlow 是「管企微私域生意」**——客户从哪来、谁在跟、AI 怎么说、人是否发出、有没有成交，全在同一套系统里。

---

## 官网对比表（可直接贴落地页）

| 维度 | 扣子 Coze | ZhiFlow |
|------|-----------|---------|
| 产品类型 | AI 智能体 / 工作流平台 | 企业微信私域增长与 CRM |
| 核心问题 | 如何快速做一个会聊天的 Bot | 如何管好客户资产并完成转化复购 |
| 企微深度 | 可作为发布渠道之一 | 原生：活码、同步、回调、侧边栏、继承 |
| 客户与跟进 | 弱（非 CRM） | 客户、阶段、标签、归属、跟进时间线 |
| 营销增长 | 插件/工作流为主 | 活码归因、裂变、群发、自动化 SOP、广告 ROI |
| AI 用法 | 对话与编排中心 | AI 意向分、话术推荐、收件箱草稿 + **人工审核后发** |
| 成交与售后 | 需自建或对接 | 订单、服务工单、收件箱转工单（闭环） |
| 团队与合规 | 应用级权限 | 多租户、RBAC、审计日志 |
| 部署 | 字节云为主 | 支持私有化部署，数据在客户服务器 |
| 适合谁 | 要快速上线问答/活动 Bot | 销售团队日常在企微里获客跟单复盘 |

**推荐组合**：复杂对话用扣子；客户数据、触达、审核、工单与看板用 ZhiFlow，API 对接即可。

## 销售话术（3 段）

### 电梯演讲（30 秒）

「我们不是另一个聊天机器人平台。ZhiFlow 是给用企业微信做私域的销售团队用的：客户自动进系统、意向分告诉你先跟谁、AI 在收件箱里写草稿、你点一下才发给客户，成交和售后还能转工单。扣子适合搭 Bot，我们适合把私域当成公司资产来经营。」

### 客户问「和扣子有什么区别？」

「扣子解决的是 **AI 能力怎么搭**；我们解决的是 **客户怎么管、怎么跟、怎么成交**。如果您只要官网客服，扣子可能更快；如果您要销售在企微里每天用的系统——活码、群发、继承、数据看板、审核发消息——那是我们的场景。很多客户两个一起用：扣子想，ZhiFlow 记、管、发、算。」

### 客户问「能不能私有化？」

「可以。后端可部署在您自己的服务器，客户与跟进数据在您的库里，满足数据不出域和审计要求。扣子以云服务为主，我们是 **私域运营系统 + 可选私有化**，不是单纯调大模型 API。」

## 投标 / 方案应答（要点清单）

**1. 建设目标匹配**

- 本项目若要求「企微客户统一管理、销售跟进、营销触达、数据复盘」，ZhiFlow 与建设目标一致；若仅要求「智能问答演示」，可补充扣子类平台作为对话层，ZhiFlow 作为业务与数据层。

**2. 功能对照（应答表常用句）**

| 招标要求（示例） | ZhiFlow | 扣子 |
|------------------|---------|------|
| 企微客户同步与归属 | 支持 | 不覆盖 |
| 渠道活码与来源统计 | 支持 | 不覆盖 |
| 群发与自动化 nurture | 支持 | 需二次开发 |
| 销售权限与操作审计 | 支持 | 弱 |
| 智能客服对话 | 支持（可对接） | 强 |
| 复杂 Agent 工作流 | 可 API 对接 | 强 |
| 私有化部署 | 支持 | 以云为主 |

**3. 风险与合规**

- 对外话术经 **人工审核后发送**，降低 AI 乱答与合规风险。
- 基于企业微信官方接口，非协议外挂。
- 高危操作（导出、删客户、群发）可审计。

**4. 实施建议（显得专业）**

- 一期：企微接入、客户同步、活码、基础自动化、仪表盘。
- 二期：统一收件箱、AI 草稿审核、知识库、SLA。
- 三期：订单/工单、公域 Webhook、与扣子/其他 LLM 的 API 编排（可选）。

**5. 不贬低竞品的收尾句**

「建议采用 **业务系统（ZhiFlow）+ 对话引擎（可选扣子/自建模型）** 的分层架构，避免把客户资产和对话能力绑死在单一 SaaS 上。」

## 对内：什么时候不推荐我们

- 只要 H5 活动问答、无 CRM、无销售团队 → 优先扣子或轻量 Bot。
- 强依赖飞书/海外渠道、企微不是主阵地 → 评估渠道优先级。
- 客户要的是「最强 Agent 实验室」而非「销售运营系统」→ 诚实推荐扣子 + 我们作数据层。

## 对内：什么时候必推我们

- 企微已是主获客与跟进渠道。
- 强调客户归属、离职继承、撞单、审计。
- 要 ROI、活码、裂变、群发与漏斗数据。
- 要 AI 辅助但 **必须人审后发**。
- 招标写明私有化或数据本地化。
